# Recovery Portal

> Self-hosted rehabilitation programme management for physiotherapy, occupational therapy, and recovery clinics.

[![License: PolyForm Small Business](https://img.shields.io/badge/license-PolyForm--SB--1.0.0-blue)](LICENSE)

Recovery Portal gives your clinic a branded, professional way to deliver rehab programmes to clients — exercise videos, milestone tracking, two-way comments, and progress dashboards — without paying a SaaS provider a per-client monthly fee.

**Run it on your own infrastructure. Brand it as your own. Own your clients' data.**

---

## Features

- 🏥 **Per-client rehab programmes** — assign types, stages, milestones, exercises
- 🎥 **Video delivery** — upload, organise, stream privately to clients
- ✅ **Progress tracking** — milestones, video views, completion percentages
- 💬 **Per-assignment conversations** — clinician ↔ client comments scoped to each assignment
- 📊 **Activity dashboards** — recent activity, stale clients, adherence reports
- 🎨 **Full white-label** — your logo, colours, name, footer, custom email identity
- 🔐 **TOTP 2FA** — enforceable for admins/clients, with trust-device and recovery codes
- 📱 **PWA + Web Push** — installable on iPad / Android, real-time notifications
- 🌓 **Dark mode** — system-aware, per-user preference
- 🔒 **Self-hosted** — your server, your data, no third-party SaaS

## Quick start (Docker)

```bash
# 1. Clone the repo
git clone https://github.com/DoFoT9/recovery-portal.git
cd recovery-portal

# 2. Generate secrets and start
cp .env.example .env.local
./scripts/init-secrets.sh   # generates AUTH_SECRET and TOTP_ENCRYPTION_KEY
docker compose up -d

# 3. Open http://localhost:3000/setup to create your first admin
```

That's it. Open the URL in your browser to complete first-run setup.

## Quick start (from source)

```bash
git clone https://github.com/DoFoT9/recovery-portal.git
cd recovery-portal
cp .env.example .env.local
./scripts/init-secrets.sh
npm install
npm run migrate:all
npm run build
npm run start
```

Open http://localhost:3000/setup and you're off.

## Configuration

All config lives in `.env.local`. See `.env.example` for every option with comments.

Required:
- `AUTH_SECRET` — 32-byte hex for session JWTs (auto-generated by `init-secrets.sh`)
- `TOTP_ENCRYPTION_KEY` — 32-byte hex for TOTP secret encryption (auto-generated)

Optional:
- `COOKIE_SECURE=true` in production (HTTPS only)
- `VAPID_PUBLIC_KEY` / `VAPID_PRIVATE_KEY` / `VAPID_SUBJECT` for Web Push notifications
- `DATA_DIR` — override the data directory (default: `./data`)

## Backup & restore

```bash
./scripts/backup.sh                                              # creates backups/recovery-portal-YYYYMMDD-HHMMSS.tar.gz
./scripts/restore.sh backups/recovery-portal-20260623-143000.tar.gz
```

Includes the SQLite DB, all uploaded videos, branding assets, and metadata.

## Upgrading

```bash
# Docker
git pull origin main
docker compose pull && docker compose up -d

# Or from source
git pull origin main
npm install
npm run migrate:all
npm run build
npm restart
```

Migrations are idempotent — safe to run multiple times.
